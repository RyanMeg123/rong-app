# AGENTS

## Stich MVP

When the user asks to `stich mvp`, `stitch mvp`, `搭 MVP`, or reduce scope to a first shippable version:

- optimize for one persona, one golden path, and one success signal,
- cut non-essential features aggressively,
- prefer mocked or static data when it proves the loop faster,
- state assumptions as `[推断]`,
- end with the next 3 concrete actions.

## Workflow Routing

Use the installed workflow skills when they fit:

- `s1-idea-validator` for weak or unvalidated ideas
- `s2-prd-drafter` for MVP scoping and acceptance criteria
- `s4a-s5-ui-designer` for screen direction and visual structure
- `s4b-s6-backend-architect` for backend shape and API contracts
- `s8-expo-rn-frontend` for Expo + React Native implementation
- `s9-node-backend-dev` for backend implementation
- `s10-api-integration` for frontend/backend connection
- `s11-qa-testing` for delivery verification
- `s12-s14-release` for release handoff

## Output Shape

Prefer a concise structure:

1. `MVP Slice`
2. `Must Have Now`
3. `Cut for Later`
4. `Workflow Skills to Use`
5. `Next 3 Actions`
6. `Key Risks`
